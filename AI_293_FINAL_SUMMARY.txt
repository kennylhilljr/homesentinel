KEY FEATURES IMPLEMENTED
================================================================================

EVENT CREATION:
  - New device detected → event + alert created automatically
  - Device goes offline → disconnect event + offline alert
  - Device comes online → connect event
  - All within same poll cycle (< 60 seconds)

API ENDPOINTS:
  - GET /api/events - Query all events with filters
  - GET /api/events/{device_id} - Device-specific events
  - GET /api/events/alerts - Query all active alerts
  - GET /api/events/alerts/{device_id} - Device-specific alerts
  - POST /api/events/alerts/{id}/dismiss - Dismiss alerts
  - GET /api/events/stats - Event statistics

FILTERING & PAGINATION:
  - Filter by device ID
  - Filter by event type (connected/disconnected/new_device)
  - Filter by date range (start_date/end_date)
  - Paginate with limit (10-1000) and offset
  - Total count with each response

UI FEATURES:
  - Tab interface (Events | Alerts)
  - Device dropdown filter
  - Event type dropdown filter
  - Date range pickers
  - Clear filters button
  - Pagination with Previous/Next buttons
  - Relative timestamps ("2h ago", "just now")
  - Color-coded event badges
  - Dismiss alert button

RETENTION CLEANUP:
  - Runs daily at 2 AM (configurable via RETENTION_CLEANUP_HOUR)
  - Deletes events older than 90 days (configurable via RETENTION_DAYS)
  - Deletes dismissed alerts older than retention period
  - Logs cleanup results (rows deleted, date range)
  - Provides retention statistics endpoint
